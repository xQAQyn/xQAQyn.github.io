HTTP协议

Posted on 2024-08-11 Edited on 2024-11-09

概述

HTTP：Hyper Text Tranfer Protocol（超文本传输协议）

特点：

  • 基于TCP协议：面向连接、安全
  • 基于请求-响应模型：一次请求一次响应
  • 无状态协议：对事务处理没有记忆能力，每次请求-响应独立
    • 缺点：多次请求无法共享数据
    • 优点：性能

请求协议

请求报文构成：

  • 请求行：
    • 请求方式：GET、POST……
    • 资源路径：从/开始，包含参数
    • 协议：例如HTTP/1.1
  • 请求头：若干行组成，格式均为key: value，常见请求头如下
    • Host：请求主机名
    • User-Agent：浏览器版本
    • Accept：表示请求的资源类型，如text/*、image/*，还可以使用*/*表示任意类型
    • Accept-Language：代表偏好语言
    • Accept-Encoding：表示接受的压缩类型，例如gzip、deflate等
    • Content-Type：请求体的数据类型
    • Content-Length：请求体的大小（字节）
  • 请求体：POST请求存放参数的部分

注意：GET请求携带的参数会作为路径参数传递

响应协议

响应报文构成：

  • 响应行：
    • 协议
    • 状态码：
      • 1xx：临时状态码，只表示响应被成功接收
      • 2xx：成功状态码
      • 3xx：重定向，让客户端再发起一次请求以完成处理
      • 4xx：客户端请求错误
      • 5xx：服务器处理错误
    • 描述
  • 响应头：key: value，常见响应头
    • Content-Type：响应体的类型
    • Content-Length：响应体长度
    • Content-Encoding：压缩方式
    • Cache-Control：指示客户端如何缓存，例如max-age=300则表示最多缓存300秒
    • Set-Cookie：告诉浏览器为当前页面所在的域设置cookie
  • 响应体
