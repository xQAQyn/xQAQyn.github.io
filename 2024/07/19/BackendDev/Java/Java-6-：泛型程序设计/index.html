限制与局限性

由于类型擦除，Java的泛型会有很多的局限性：

  • 不能使用基本类型作为类型参数：例如不能使用Pair<double>而需要使用Pair<Double>，原因就在于类型擦除，因为类型擦除会将其改变为Object，而基本类型无法转换为Object

  • 运行时的类型查询不能判断是否为某泛型：由于会执行类型擦除，所以类似于以下判断对象是否属于泛型会产生错误：

  • 不能创建参数化类型的数组：无法创建参数化类型的数组，但参数化类型数组这一类型是正确的：

泛型的继承规则

  • 无论类型T和类型S是什么关系，Pair<T>和Pair<S>都没有关系
  • Pair<T>是原始类型Pair的一个子类型
  • 泛型类可以像普通类一样拓展或实现其它类
