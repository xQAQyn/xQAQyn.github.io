Posted on 2024-07-12 Edited on 2024-07-13

面向对象的程序设计

类之间的关系：

  • 依赖：“use-a”关系，一个类的方法使用或操纵另一个类的对象
    • 依赖会导致类之间的耦合，现代软件工程认为应当尽量减少类之间的依赖
  • 聚合：“has-a”关系，一个类包含另一个类的一些对象
  • 继承：“is-a”关系，一个更特殊类与一个更一般的类的关系

静态的常量是一个更常用的用法，例如Math.PI、System.out都为其静态常量

静态方法：是不在对象上执行的方法，例如Math.pow(x, a)计算xax^axa，并不使用任何Math类的对象。可以理解为静态方法就是不含隐式参数的方法。main方法就为一个静态方法。

注意：访问静态字段不属于访问对象，因为静态对象是一个类的字段

静态方法还有一个常见用途是通过静态工厂方法来构造对象，比起构造器主要有如下好处：

  • 可以命名工厂方法区分不同的构造方式
  • 可以改变构造对象的类型（例如返回其子类）

包

包名：

  • 一般通过域名的逆序形式作为包名，例如域名是horstmann.com，则包名为com.horstmann
  • 然后可以追加一个工程名，例如com.horstmann.corejava
  • 其后还可以追加具体的类名，例如com.horstmann.corejava.Employee

导入包：

本地的包通过路径访问，路径以.分隔各个层级，使用相对路径，根由编译指令指定

访问权限修饰符：

  • public：可以由任意类使用
  • private：只能由定义它们的类使用
  • 没有修饰符：可以被同一包内的所有方法访问
