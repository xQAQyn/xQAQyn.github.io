Posted on 2024-07-18 Edited on 2024-07-19

异常

在Java中所有的异常都是派生于Throwable类的一个实例，有如下层次结构：

我们需要关注的是程序的异常，包含RuntimeException和其它异常两个分支。

派生于RuntimeException的异常包括：

  • 错误的强制类型转换
  • 数组访问越界
  • 访问null指针

不是派生于RuntimeException的异常包括：

  • 视图超越文件末尾读取数据
  • 视图打开不存在的文件
  • 视图根据给定字符串查找Class对象，但对于的类并不存在

异常分类：

  • 非检查型：包括Error和RuntimeException
  • 检查型：其它异常

对于非检查型的异常我们应当避免，而检查型异常则可以设置异常处理器

断言

assert有两种形式：

如果condition不满足的话，就会抛出一个AssertionError异常，如果后面还有表达式，那么表达式将传入AssertionError构造器，并转换成一个消息字符串。

断言是默认关闭的，如果开启需要在运行程序时加上-enableassertions或-ea参数。一般只在开发和测试阶段使用断言。

其次，由于断言失败会直接导致程序终止，所以断言一般用于测试以及程序员进行自我检查，不用于实际生产环节
