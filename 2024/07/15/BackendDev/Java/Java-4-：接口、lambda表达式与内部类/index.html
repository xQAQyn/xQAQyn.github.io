默认方法直接可能产生冲突，例如在超类或另一个接口定义了同名的方法，此时Java的解决规则为：

  1. 超类优先：如果超类提供了一个具体方法，同名且有相同参数类型的默认方法被忽略
  2. 接口冲突：如果两个接口提供了同名且参数类型相同的方法（至少有一个提供了默认实现），必须覆盖这个方法解决冲突

如果想要在其它部分代码中实现真正的深拷贝，那么需要：

  • 实现Cloneable接口：Cloneable是一个标记接口，它不包含任何方法，唯一的作用是允许在类型查询中使用instanceof
  • 重新定义clone方法，并指定为public

变量作用域

lambda表达式中可能含有自由变量，即既不是参数也没有进行定义的变量，例如下面lambda表达式中的text变量：

text变量为repeatMessage的参数，而不在lambda表达式中定义。这里就会出现一个问题，lambda表达式的内容可能在repeatMessage执行结束后才开始执行，此时该参数已经不存在了。

所以Java提供了一套机制，让lambda表达式将自由变量存储下来，这个过程称为捕获。在Java中需要确保捕获的值是明确定义的，且只能捕获值不会改变的变量，即事实final变量（初始化后就不再赋新值）。

lambda表达式的体语气嵌套块有相同的作用域，这意味着在lambda表达式中声明的变量不可与其嵌套块中的变量重名。

内部类

内部类是在一个类的定义中定义另一个类，主要有两个好处：

  • 对同一个包中的其它类隐藏
  • 内部类的方法可以访问定义这个类的作用域中的数据，包括私有数据
