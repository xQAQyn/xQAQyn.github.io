链路层概述

链路层提供的服务包括：

  • 成帧：使用链路层帧封装数据报
  • 链路接入：媒体访问控制(Medium Access Control, MAC)协议规定帧在链路上的传输。对多点共享单个广播链路的情况，MAC协议会协调多个节点的帧传输
  • 可靠交付：链路层也通过确认和重传机制实现可靠交付，在高差错率的链路上可以避免使用运输层和应用层进行实现
  • 差错检测和纠正

链路层主体部分在网络适配器（也被称为网络接口卡，网卡，NIC）中实现，

多路访问链路和协议

链路分为：点对点链路、广播链路

广播链路存在多路访问冲突的问题，即多个节点同时向链路发送数据，数据产生碰撞，数据丢失。这需要通过多路访问协议解决。多路访问协议主要分为以下三类：

  • 信道划分协议
  • 随机接入协议
  • 轮流协议

信道划分协议

时分多路复用(TDM)：将时间划分为时间帧，然后进一步将每个时间帧划分为 NNN 个时隙，链路相连的 NNN 个设备每个设备获取一个时隙。优点是消除了碰撞且公平。缺点是不发送的节点也会占用时间。

频分多路复用(FDM)：将 RRR bps的带宽划分为不同频段，每个频段的带宽为 RN\frac{R}{N}NR​ bps，每个设备使用一个信道。优缺点与TDM相同

码分多址(CDMA)：CDMA给每个节点分配了一种不同的编码。如果编码选择合适，可以实现数据间不受干扰，共同传输。

随机接入协议

时隙Aloha

需求：

  • 一个时隙能够传递一个数据帧
  • 各个节点同步，都只在时隙开始时开始传输
  • 一个时隙中若出现帧的碰撞，则所有节点可以检测到

节点的行为是：

  • 节点有帧要发送，则会在下一个时隙开始时开始传输这一帧
  • 传输的帧若没发生碰撞，则视为成功
  • 如果出现碰撞，则节点以 ppp 的概率在后续时隙中重传这一帧，直到帧被传出

原始Aloha是没有需求2的版本

载波侦听多路访问(CSMA)

规则：

  • 载波侦听：如果检测到另一个帧在信道上传播，则等待直到一小段没有帧传输，才开始传输自己的帧

具有碰撞检测的载波侦听多路访问(CSMA/CD)

规则：

  • 载波侦听：如果检测到另一个帧在信道上传播，则等待直到一小段没有帧传输，才开始传输自己的帧
  • 碰撞检测：传输时持续侦听信道，如果检测到碰撞，则停止传输，并等待一个随机时间后回到上一条规则进行传输
    • 二进制指数退避算法：在第 kkk 次重传时，随机取 [0,2k−1][0,2^k-1][0,2k−1] 之间的一个数作为等待时间

争用期：以太网端到端的往返时延上限 2τ2\tau2τ ，经过了这一事件就可以确认不发生碰撞

轮流协议

轮询协议：指定一个节点为主节点，以循环方式轮询每个节点，告诉其能够发送的帧的最多数量。缺点是引入了轮询时延。

令牌传递协议：将一个称为令牌的小的特殊帧以固定次序进行交换。得到令牌的节点如果要发送，则持有令牌，否则直接转发给下一个节点。缺点是单个节点故障会导致令牌传递出现问题。

交换局域网

MAC地址

组成：48bit

作用：链路层标识每个网络适配器（位于网络适配器ROM，不可更改，与硬件本身绑定）

地址解析协议(ARP)

作用：映射子网内的IP地址到MAC地址，映射格式为<IP; MAC; TTL>，TTL为有效期(典型值为20分钟)

ARP运行流程如下：假设222.222.222.220要向222.222.222.222发送数据报

  • 发送方构造ARP查询分组，其中目的地的MAC地址被设为MAC广播地址（全1）
  • 对应IP的接收方收到ARP请求后，构造ARP应答，发送给发送方
  • 发送方根据接收到的MAC地址发送，同时缓存这一映射到ARP表中

ARP本身是即插即用的，无需配置

以太网

以太网帧的结构如下：

  • 前同步码：8字节，前7字节为10101010，最后1字节为10101011。用于调整时钟保持同步
  • 目的地址：6字节，目的适配器的MAC地址
  • 源地址：6字节，源的MAC地址
  • 类型字段：2字节，允许以太网复用多种网络协议
  • CRC：检错字段

以太网提供的服务是无连接、不可靠的服务

以太网使用CSMA/CD，特殊的是：

  • 以太网使用 51.2μs51.2\mu s51.2μs 为争用期长度，对10Mbps的以太网，争用期内可以发送512bit，64字节。所以64字节内不碰撞就不会发生碰撞
  • 基于上一点，发生碰撞一定在前64字节以内。由于碰撞立即停止，所以将小于64字节的帧视为无效帧

交换机

交换机是存储-转发设备，在MAC层实现了LAN互连。其概念结构如下

工作原理：

  • 不断监听各接口是否有信号
  • 收到无差错的帧则缓存，否则丢弃
  • 若收到的帧目的MAC地址属于另一网段，则通过站表转发至对应接口
  • 若收到的帧目的MAC为同一网段，则不转发
  • 交换机不修改所转发帧的源地址

优势：过滤通信量、扩大局域网的物理范围、提高可靠性、互连不同物理层、不同MAC子层和不同速率的局域网

缺点：增加时延、缓存区满时出现帧丢失、可能出现广播风暴

与集线器区别：

  • 集线器只是将网络覆盖距离简单延长，实现在物理层；交换机将LAN覆盖距离延长，理论上可以做到无限延长，具体实现在MAC层
  • 集线器仅具有简单的信号整形和放大功能，交换机提供信号的存储/转发、数据过滤、路由选择等能力
  • 集线器是硬设备，交换机有软硬件

交换机具有透明性，即对各站来说是不可见的

交换机选路原理：接收到帧时保存其来源的MAC已经对应端口，若不知道MAC对应端口在哪则广播

支撑树算法：交换机互联后，自动断开某些接口，形成最小生成树

  • 选择一个根，然后运行最短路径

交换机vs路由器：

  • 都是存储转发设备
  • 交换机为链路层设备，路由器为网络层设备
  • 路由器维护路由表，实现路由算法；交换机维护交换表，实现MAC地址过滤、学习算法
功能 集线器 路由器 交换机
流量隔离 无 有 有
即插即用 有 无 有
优化选路 无 有 无
直通交换 有 无 有

虚拟局域网VLAN

VLAN为基于端口的虚拟局域网，通过交换机管理软件完成。

构建方式：

  • 基于交换机的接口号
  • 基于MAC地址
  • 基于IP地址

VLAN好处：

  • 流量隔离：VLAN内部的帧只能发送到VLAN内部
  • 动态的成员管理：端口可以在VLAN间动态分配

跨VLAN的转发：通过路由wan’c
