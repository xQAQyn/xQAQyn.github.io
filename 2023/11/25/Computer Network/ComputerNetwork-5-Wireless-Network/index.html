无线网络概述

无线网络基本构件

  • 无线主机：本身可能是固定的或移动的，能运行程序，例如便携手机
  • 无线链路：用于连接无线主机和基站，使用多址访问协议
  • 基站：用于连接无线网络，负责向覆盖范围内的主机发送和接收分组

分为两种模式：

  • 基础设施模式：使用预先建立的、能覆盖一定地理范围的一批固定基站，主机通过基站接入有线网络。
    • 关联：无线主机位于某个基站的无线通信覆盖范围内时，该主机使用该基站中继它与更大网络之间的数据
    • 切换：当移动主机移动范围超出了一个基站的覆盖范围而到达另一个基站的覆盖范围后，它将改变其接入更大网络的连接点
  • Ad hoc网络：没有固定基础设施的自组网络。无基站，节点仅仅能在其覆盖范围内向其它节点传送数据，节点之间的通信组成临时网络，在内部进行选路和地址分配

无线链路和网络特性

无线链路特征：

  • 信号强度衰减
  • 有来自其它源的干扰
  • 多径传播
  • 比特差错比有线链路更常见

无线网络引入的新问题：

  • 隐藏终端问题：由于信号衰减和障碍物的存在会导致终端之间不可见。例如A和B可见，B和C可见，但A和C有可能不可见，这样如果AC在B处发生碰撞，AC无法检测到该碰撞

差错处理

无线链路中的差错处理方案：

  • 使用CRC进行帧校验
  • 使用ARQ协议进行重传

描述无线链路的质量：信噪比(SNR)、比特差错率(BER)

  • 对给定的调制方案，SNR越高，BER越低
  • 对固定的SNR，具有较高比特传输率的调制方案有较高的BER

码分多址(CDMA)

CDMA是一个信道划分协议，在WLAN和蜂窝技术中应用广泛。

CDMA允许每个用户在同样的时间内用相同的频带进行通信，但每个用户使用精心挑选的不同码型，互不干扰。

工作流程：

  • 每个用户获得唯一的一个 mmm bit码片，不同用户的码片正交
  • 发送比特1时，发送这一码片；发送比特0时，发送码片的反码

802.11 WLAN

802.11协议簇：

  • 802.11b：工作在2.4-2.485GHz频段上，最高速率11Mbps
  • 802.11a：工作在5.1-5.8GHz频段上，最高速率54Mbps
  • 802.11g：工作在2.4-2.485GHz频段上，最高速率54Mbps
  • 802.11n：2.4-2.485GHz、5.1-5.8GHz，多天线，单流最高速率150Mbps，多流最高速率600Mbps
  • 802.11ac：5.1-5.8GHz，单流最高433Mbps@80MHz、866Mbps@160MHz，多流1.73Gbps、3.47Gbps
  • 802.11ax（第六代WiFi）：单流速率提升至1.2Gbps， 8×88\times 88×8 模式下提供高达9.6Gbps传输速率
    • 支持较窄的子载波间隔以及增加的符号持续时间，有利于确保信号的健壮性，能覆盖更远范围
    • 引入正交频分多址(OFDMA)，同时支持2.4GHz和5GHz频段，创造出更多可用数据通道，解决拥堵
    • 引入LTE领域组网特性，让信号抗干扰能力大幅提升

802.11体系结构：

  • 无线终端通过基站(AP)进行通信，一个基站的服务范围为一个基本服务集(BSS)
  • 一个BSS包括：基站(Ad hoc模式下没有)、无线终端

802.11b

802.11b信道划分：2.4GHz-2.485GHz，供85MHz，划分为11个部分重叠的信道。当两个信道中间相隔4个及以上的信道时，信道无重叠，可同时工作

主机关联AP的过程：

  1. 每个AP周期性发送信标帧，包括AP的SSID(服务集标识)和MAC
  2. 主机对11个信道进行扫描，获取所有可用的AP信标帧
  3. 主机选择一个进行关联，加入其子网
  4. 主机向关联AP发送DHCP发现报文，获取IP地址
  5. 可能需要身份鉴别

802.11MAC协议：CSMA/CA

发送方流程：

  1. 侦听信道，直到信道闲置了 DIFSDIFSDIFS 秒，直接传输整个帧，不做冲突检测
  2. 如果侦听到信道忙，选择一个随机的退避值(二进制指数退避)，等待对应时间
  3. 等待结束后如果信道空闲就发送数据
  4. 如果收到确认，该次发送结束；如果没有收到确认，再次选择随机的退避值(二进制指数退避)，回到第二步

接收方：收到帧后等待 SIFSSIFSSIFS 秒后返回ACK

不采用冲突检测的原因：

  • 碰撞检测要求站点同时具有发送和接收的能力
  • 802.11适配器，接收信号的强度远小于发送信号的强度，会被淹没无法检测到
  • 如果一定要实现，硬件代价巨大
  • 由于隐蔽终端问题和信号衰减，无法检测到所有的碰撞

为了避免长的数据帧冲突，可以使用冲突避免，“预约”信道进行发送：

  1. 发送方在发送前使用CSMA协议发送一个短请求RTS(request-to-send)帧给AP
  2. AP广播一个允许发送CTS(clear-to-send)帧响应RTS
  3. 所有节点侦听到CTS，推迟发送
  4. 发送结束后，AP广播ACK，其它节点结束推迟发送的状态

速率自适应

基站和终端之间的传输速率会随终端移动和SNR的变化而智能调整

  • 当终端原理基站时，SNR减小、BER增大
  • BER增大到一定程度时，将速率切换到一个较低水平，保障较低的BER

移动管理

移动节点需要保持一个不变的地址，这样才能让上层对移动性不可见。

定义：

  • 永久地址(MA)：移动设备保持不变的IP地址
  • 归属网络：移动设备永远的“家”，不变的网络。永久地址属于归属网络中
  • 归属代理(HA)：当移动用户在远程时，代表移动节点执行移动管理功能的实体
  • 被访问网络：移动设备当前所在网络
  • 转交地址(COA)：在被访问网络中，移动设备的地址
  • 外部代理：在被访问网络中帮助移动节点

移动性的两种解决方法：

  • 让路由器处理：路由器直接通过路由表交换方式向邻居通告移动节点的永久地址

    • 缺点：可扩展性差

  • 让终端处理：

    • 间接路由：通信者如果需要与移动用户通信，归属代理截获这些数据报，然后通过COA转发给外部代理，外部代理转发给移动用户

    • 直接路由：通信者通过归属代理获得移动用户的外部地址，然后直接发送给移动用户

直接路由引入了新问题：

  • 需要一个移动用户定位协议，一遍通信这向归属代理查询COA
  • 移动用户移动到一个新的外部网络后，之前的COA没有被及时更新

锚外部代理解决了这一问题，其修改为：

  • 当移动节点到达一个信外部网络后，移动节点向新的外部代理注册，并且新的外部代理向锚外部代理提供移动节点的新COA
  • 如果锚外部代理收到一个发往已经离开的移动节点的数据报，它可以使用新的COA重新封装发送
